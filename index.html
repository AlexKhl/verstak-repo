    This is a repository of "Verstak" project.

    Project maintained by AlexKhl Hosted on GitHub Pages — Theme by mattgraham

    What is Verstak?

    This automatically Drupal-site building system allowing you to create sites from template that you previously created. You can include node types, views, blocks, imagecache patterns in your site before building, and after building Verstak also allows you to view different versions of sites in the design, build, each located in a directory named build date of creation, which allows you to track changes to the site by changing the template files for views, blocks etc.

    Where to start?

    To install the system requires a system of Linux (Ubuntu in priority).

    You must have the following ruby 1.8.7 and gem's on your Linux:

    builder (3.0.0)
    childprocess (0.2.1)
    cucumber (1.0.2)
    diff-lcs (1.1.2)
    ffi (1.0.9)
    gherkin (2.4.11)
    json (1.5.3)
    json_pure (1.5.3)
    rake (0.9.2)
    rspec (2.6.0)
    rspec-core (2.6.4)
    rspec-expectations (2.6.0)
    rspec-mocks (2.6.0)
    rubyzip (0.9.4)
    selenium-webdriver (2.4.0)
    term-ansicolor (1.0.6)
    watir-webdriver (0.3.2)
    You need to have installed drush (sudo apt-get install drush) http://drush.ws
    For additional information learn Readme file.

    Next:

    1) Download the Verstak archive.
    2) Extract the Verstak archive to a working directory of server.
    3) Copy the file template.config.rb and rename it to config.rb.
    4) Define in the config.rb settings for the future system: the user database, the db_user's password, the name of the first Drupal-user and password for the site, the directory of destination (where you store the site example: /var/www/verstak), these and other parameters are listed in the default.config.rb file, in the file config.rb you will override them.
    5) In the command line you should go to that directory and run the following command

    $ rake

    6) Drupal-site is ready for using
